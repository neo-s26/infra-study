# AZ-900 最後の1時間で見直す横断チェック

## 使い方

- 各章5問ずつ、合計40問
- まず答えを隠して解く
- 迷った問題だけ、該当章の試験型ガイドに戻る
- 最後の1時間では「完璧に覚える」より「混同を潰す」ことを優先する

### Q1. クラウドとオンプレミスの違いとして正しいものは？
**答え：C**

クラウドは、クラウドサービスプロバイダーが保有・管理するリソースを利用する。  
オンプレミスは、自社でサーバーやネットワーク機器を設置・管理する形態。

### Q2. CAPEXとOPEXの説明として正しいものは？
**答え：B**

CAPEXはサーバー購入などの設備投資。  
OPEXは利用量に応じて支払う運用費。  
クラウドではCAPEXを減らし、OPEX中心にしやすい。

### Q3. 可用性ゾーンの説明として正しいものは？
**答え：B**

可用性ゾーンは、同一リージョン内のデータセンター障害に備える仕組み。  
リージョン全体の障害対策は、geo分散やリージョンペアの考え方。

### Q4. IaaS / PaaS / SaaSの説明として正しいものは？
**答え：C**

IaaSは仮想マシンなどのインフラを利用する。  
PaaSはアプリ実行基盤を利用する。  
SaaSは完成済みアプリを利用する。

### Q5. ハイブリッドクラウドの例として正しいものは？
**答え：C**

ハイブリッドクラウドは、オンプレミスやプライベートクラウドと、パブリッククラウドを組み合わせる構成。

### Q1. Azure Policyの説明として正しいものは？
**答え：A**

Azure Policyは、許可リージョン、VMサイズ、タグ必須などのルールを適用・評価するサービス。

### Q2. ポリシー定義とイニシアチブ定義の違いとして正しいものは？
**答え：B**

ポリシー定義は個々のルール。  
イニシアチブ定義は複数ポリシーをまとめた集合。

### Q3. 削除ロックと読み取り専用ロックの違いとして正しいものは？
**答え：A**

削除ロックは削除を禁止するが、作成や更新は可能。  
読み取り専用ロックは閲覧のみ許可し、作成・更新・削除を禁止する。

### Q4. Azure Advisorの説明として正しいものは？
**答え：A**

Azure Advisorは、コスト、セキュリティ、信頼性、パフォーマンス、運用効率の観点で推奨事項を提示する。  
強制力はない。

### Q5. メトリックとログの違いとして正しいものは？
**答え：A**

メトリックはCPU使用率などの数値データ。  
ログは「誰が何をしたか」「どんなイベントが発生したか」などのテキストデータ。

## 最後の横断チェック：混同しやすい超重要ペア

| 比較 | 覚え方 |
|---|---|
| 認証 / 認可 | 認証は「誰か」、認可は「何をしてよいか」 |
| Azure AD / AD DS | Azure ADはクラウド、AD DSはオンプレミス |
| IaaS / PaaS / SaaS | IaaSはVM、PaaSは実行基盤、SaaSは完成アプリ |
| 可用性ゾーン / リージョンペア | ゾーンは同一リージョン内、ペアは別リージョン |
| 可用性セット / 可用性ゾーン | セットはラック・メンテ対策、ゾーンはデータセンター対策 |
| VNet / サブネット | VNetは大枠、サブネットは分割範囲 |
| サイト間VPN / ExpressRoute | VPNはインターネット、ExpressRouteは専用線 |
| DNSゾーン / プライベートDNSゾーン | DNSゾーンは外向け、プライベートDNSはAzure内部向け |
| LRS / ZRS / GRS | LRSは1DC、ZRSは複数ゾーン、GRSは別リージョン |
| ホット / クール / アーカイブ | 頻繁・低頻度・長期保管 |
| Azure Policy / RBAC | Policyはルール、RBACは権限 |
| Azure Policy / リソースロック | Policyは構成ルール、ロックは操作禁止 |
| Azure Advisor / Azure Monitor | Advisorは推奨、Monitorは監視 |
| Service Health / Resource Health | Service HealthはAzureサービス、Resource Healthは個別リソース |
| 料金計算ツール / コスト分析 | 料金計算は利用前、コスト分析は利用中・利用後 |
