# 第3章 Azureの管理｜試験型ガイド

## 1. この章で問われること

この章は、Azureリソースを管理するための方法と、コスト管理の基本を理解する章。

試験では、細かい操作手順よりも次の観点が問われやすい。

- Azure Resource Managerの役割
- Azure portal / Azure PowerShell / Azure CLI / Cloud Shell / Azure Mobile Appsの違い
- ARMテンプレートを使うメリット
- Azure Arcで管理できる対象
- Azureコストに影響する要素
- 料金計算ツールとTCO計算ツールの違い
- コスト分析、予算アラート、タグの使い方

---

## 2. 5分で見直す要点

### Azure Resource Manager

Azure Resource Manager、ARMは、Azureリソースへの管理操作を受け取る仕組み。

Azureでリソースを作成・変更・削除するときは、どの管理ツールを使っても最終的にARMを経由する。

試験では次のように押さえる。

- Azureリソース管理の中心
- リソース作成、変更、削除の命令を受け取る
- Azure portalからの操作もARMを経由する
- Azure PowerShellやAzure CLIからの操作もARMを経由する
- 利用者が直接意識しなくても、内部ではARMが処理している

---

### Azure portal

Azure portalは、WebブラウザーからAzureリソースを管理するGUIツール。

マウス操作で直感的にリソースを作成・構成・削除できる。

試験では次のように押さえる。

- WebベースのGUI管理ツール
- 手動操作に向いている
- リソースの一覧表示や設定変更がしやすい
- Windows、Linux、macOSからブラウザーで利用できる
- 初心者や単発作業に向いている

---

### GUIとCUI

| 種類 | 意味 | 代表例 |
|---|---|---|
| GUI | 画面を見ながらマウスなどで操作する | Azure portal |
| CUI | 文字コマンドで操作する | Azure PowerShell、Azure CLI |

GUIは手動操作、CUIはコマンド操作や自動化に向く。

---

### Azure PowerShell

Azure PowerShellは、PowerShellのコマンドレットでAzureリソースを管理するツール。

Azureを操作するには、Azure管理用のAzモジュールを使う。

試験では次のように押さえる。

- PowerShellベースの管理ツール
- コマンドレットでAzureを操作する
- Azモジュールが必要
- サインインには `Connect-AzAccount` を使う
- 仮想マシン一覧取得には `Get-AzVM` などを使う
- Windows管理者やPowerShellに慣れた人に向く

---

### Azure PowerShellの構文

PowerShellのコマンドレットは、基本的に「動詞-名詞」の形。

Azモジュールでは、名詞部分にAzが入る。

| 操作 | 例 |
|---|---|
| Azureへ接続 | `Connect-AzAccount` |
| VM一覧を取得 | `Get-AzVM` |

覚え方：

- PowerShell：`動詞-Az名詞`
- 例：`Get-AzVM`

---

### Azure CLI

Azure CLIは、Azureを操作するためのコマンドラインツール。

PowerShellではなく、Bash系のコマンドスタイルでAzureを管理する。

試験では次のように押さえる。

- コマンドラインでAzureを操作する
- Windows、Linux、macOSで利用できる
- Linuxユーザーに好まれやすい
- サインインには `az login` を使う
- 仮想マシン一覧取得には `az vm list` を使う
- Bashやクロスプラットフォーム環境に向く

---

### Azure CLIの構文

Azure CLIは、基本的に `az` から始まる。

| 操作 | 例 |
|---|---|
| Azureへ接続 | `az login` |
| VM一覧を取得 | `az vm list` |
| テーブル形式で表示 | `az vm list -o table` |

覚え方：

- Azure CLI：`az 名詞 動詞`
- 例：`az vm list`

---

### Azure Cloud Shell

Azure Cloud Shellは、Azure portalやAzure Mobile Apps内で利用できるコマンド実行環境。

PowerShellまたはBashを選んで使える。

試験では次のように押さえる。

- ブラウザーから使えるコマンド実行環境
- Azure portal内で起動できる
- PowerShellとBashを選べる
- AzモジュールやAzure CLIが標準で使える
- 事前インストールなしでコマンド操作できる
- 利用にはストレージアカウントが必要

---

### Azure Mobile Apps

Azure Mobile Appsは、スマートフォンやタブレット向けのAzure管理アプリ。

外出先でAzureリソースの状態確認や簡単な管理を行うために使う。

試験では次のように押さえる。

- iOS / Android向けの管理アプリ
- Azure portalより機能は限定される
- 小さい画面でも操作しやすい
- VMやWebアプリなどを管理できる
- リソースの正常性や状態を確認できる
- アラートを確認できる
- Cloud Shellも起動できる

---

## 3. ARMテンプレート

### ARMテンプレートとは

ARMテンプレートは、Azureリソースの作成や構成内容を定義するテンプレートファイル。

テンプレートに定義した内容どおりに、正確で一貫性のあるリソースを作成できる。

試験では次のように押さえる。

- リソース構成をコードとして定義できる
- JSON形式で記述する
- 同じ構成を何度も再現できる
- 手動操作によるミスを減らせる
- 開発環境、本番環境などで同じ構成を作りやすい
- テンプレートをバージョン管理できる
- Azure portal、Azure PowerShell、Azure CLIから実行できる

---

### ARMテンプレートが向いている場面

ARMテンプレートは、一貫した構成を何度も作りたい場合に向いている。

代表的な場面：

- 複数環境に同じ構成を作りたい
- 手動作成による操作ミスを防ぎたい
- リソース構成を標準化したい
- 何度も同じ構成を展開したい
- 構成をファイルとして管理したい
- 過去の構成に戻せるようにしたい

試験では「一貫性」「自動化」「再利用」「ミス防止」がキーワード。

---

## 4. Azure Arc

### Azure Arcとは

Azure Arcは、Azure外のリソースもAzure上で一元管理するためのサービス。

オンプレミス、他社クラウド、AzureリソースをAzure portalでまとめて管理できる。

試験では次のように押さえる。

- ハイブリッドクラウド管理に使う
- マルチクラウド管理に使う
- オンプレミスのサーバーをAzureから管理できる
- AWSなど他クラウドのリソースも管理対象にできる
- Azure外の資産をAzureリソースのように扱える

---

### Azure Arcでできること

| 用途 | 内容 |
|---|---|
| 資産管理 | IT資産情報をAzureに集約して管理 |
| サーバー監視 | Azure外のサーバーも監視対象にできる |
| 保護・更新 | Azure VM向けの管理機能をAzure外にも広げる |
| ログ管理 | ログを収集し、Azureで分析・可視化する |

Azure Arc = ハイブリッド / マルチクラウド環境の一元管理、と覚える。

---

## 5. Azureのコスト管理

### Azureコストに影響する主な要素

Azureの料金は、固定料金ではなく、リソースの種類や使い方によって変わる。

主な要素は次の3つ。

| 要素 | 内容 |
|---|---|
| リソースの種類・SKU・使用状況 | VMサイズ、利用時間、ストレージ容量、通信量など |
| リージョン | 同じリソースでもリージョンにより価格が異なる |
| 割引オプション | 予約、Azureハイブリッド特典、Spot VMなど |

---

### 仮想マシンの主な課金要素

仮想マシンの料金は、主に次の要素で構成される。

- 仮想マシン使用料金
- ストレージ料金
- データ転送料金

注意点：

- VM使用料金は、VMが起動している間に発生する
- Azure portalなどから停止すればVM使用料金は止まる
- OS上でシャットダウンしただけでは課金が続く場合がある
- ストレージ料金はVM停止中も発生する
- Azureからインターネットへの送信データは課金対象になりやすい
- リージョン間通信は課金対象になりやすい

---

### リージョンとコスト

同じ種類・同じSKUのリソースでも、リージョンによって料金が異なる場合がある。

試験では次のように押さえる。

- データセンターの施設代、電力、人件費などが地域ごとに異なる
- 本番環境では遅延や法規制も考慮する
- 開発・検証環境では安いリージョンを選ぶこともある
- コストだけでなく、性能・遅延・コンプライアンスとのバランスが必要

---

### 予約

予約は、長期間継続して利用するリソースに対する割引オプション。

試験では次のように押さえる。

- 1年または3年の利用を前提に割引を受ける
- 継続利用が決まっているリソースに向く
- 長期利用のVMなどに向く
- 短期利用や利用量が読めない場合には不向き

---

### Azureハイブリッド特典

Azureハイブリッド特典は、既存のMicrosoftライセンスをAzureに持ち込んで割引を受ける仕組み。

試験では次のように押さえる。

- Windows ServerやSQL Serverなどのライセンスを活用する
- ソフトウェアアシュアランス契約などが関係する
- ライセンス料の二重払いを避けられる
- 既にMicrosoft製品のライセンスを持つ組織に向く

---

### Azure Spot Virtual Machines

Azure Spot Virtual Machinesは、Azureの余剰リソースを割引価格で利用する仮想マシン。

ただし、Azure側のリソースが不足すると停止または削除される可能性がある。

向いている用途：

- 開発環境
- テスト環境
- 短時間で終了するバッチ処理
- 中断されても問題ない処理
- 再実行できる処理

向いていない用途：

- 本番の重要システム
- 停止すると困る業務システム
- 中断できない処理

---

## 6. コスト見積もり・管理ツール

### 料金計算ツール

料金計算ツールは、Azureで利用予定のリソースを指定して、月額料金を見積もるツール。

試験では次のように押さえる。

- Azureリソース単位で料金を見積もる
- VMサイズ、リージョン、OS、ディスク、通信量などを指定する
- 見積もりをExcel形式で出力できる
- 利用予定のリソース構成が決まっている場合に向く

---

### 総保有コスト計算ツール

総保有コスト、TCO計算ツールは、オンプレミス環境をAzureへ移行した場合のコスト削減効果を予測するツール。

試験では次のように押さえる。

- オンプレミスとAzureのコストを比較する
- サーバー本体、電力、データセンター、人件費なども考慮する
- 移行前後の総コストを比較できる
- Azure移行を検討するときに使う

---

### コスト分析

コスト分析は、現在発生しているAzure利用料金を確認するツール。

試験では次のように押さえる。

- 現在のコストを確認できる
- コストの内訳をグラフで確認できる
- 管理グループ、サブスクリプション、リソースグループ単位で集計できる
- 結果を保存してレポートとして利用できる

---

### 予算アラート

予算アラートは、Azure利用料金が設定した金額を超えた場合などに通知する仕組み。

試験では次のように押さえる。

- 予算額を設定できる
- 実際の利用金額に基づいて通知できる
- 予測金額に基づいて通知できる
- メールなどで管理者に通知できる
- 想定外のコスト発生を早期に見つけるために使う

---

## 7. タグ

### タグとは

タグは、Azureリソースやリソースグループに付ける名前と値のセット。

リソースに名札を付けるイメージ。

例：

| 名前 | 値 |
|---|---|
| ServerType | Web |
| ProjectName | ABC project |
| ResponsiblePerson | Mr.A |

試験では、タグはリソース整理やコスト分析に使うと押さえる。

---

### タグの活用場面

タグを使うと、リソースやコストを分類しやすくなる。

主な用途：

- Azure portalでリソースを絞り込む
- 特定プロジェクトのリソースだけ表示する
- 担当者ごとにリソースを分類する
- コスト分析でタグごとに料金を絞り込む
- 請求情報をプロジェクト別・部門別に整理する

---

## 8. 混同しやすい比較

### 管理ツールの比較

| ツール | 種類 | 向いている用途 |
|---|---|---|
| Azure portal | GUI | 手動操作、初学者、単発作業 |
| Azure PowerShell | CUI | PowerShellによる自動化、Windows管理 |
| Azure CLI | CUI | Bash系操作、Linux、クロスプラットフォーム |
| Azure Cloud Shell | CUI | ブラウザー上で即コマンド実行 |
| Azure Mobile Apps | モバイル | 外出先での状態確認や簡易管理 |

どの管理ツールを使っても、最終的にはAzure Resource Managerを経由してリソースが操作される。

---

### Azure PowerShellとAzure CLI

| 比較 | Azure PowerShell | Azure CLI |
|---|---|---|
| コマンド形式 | `動詞-Az名詞` | `az 名詞 動詞` |
| 例 | `Get-AzVM` | `az vm list` |
| サインイン | `Connect-AzAccount` | `az login` |
| 向いている人 | PowerShell利用者 | Bash / Linux利用者 |

---

### 料金計算ツールとTCO計算ツール

| ツール | 目的 | 向いている場面 |
|---|---|---|
| 料金計算ツール | Azure利用料金を見積もる | 作成予定のAzureリソースが決まっている |
| TCO計算ツール | オンプレミスからAzure移行時のコスト差を予測 | Azure移行の効果を比較したい |

---

### 料金計算ツールとコスト分析

| 比較 | 料金計算ツール | コスト分析 |
|---|---|---|
| タイミング | 利用前 | 利用中・利用後 |
| 目的 | 見積もり | 実際のコスト確認 |
| 対象 | 予定しているリソース | 現在発生している利用料金 |

---

### 予約、Azureハイブリッド特典、Spot VM

| 割引 | 内容 | 向いているケース |
|---|---|---|
| 予約 | 1年または3年の長期利用で割引 | 継続利用が決まっているVM |
| Azureハイブリッド特典 | 既存ライセンスをAzureに持ち込む | Windows ServerやSQL Serverのライセンス保有 |
| Spot VM | 余剰リソースを安く使う | 中断されてもよい処理 |

---

## 10. 最後に覚えるキーワード

- ARM：Azureリソース管理操作の受け取り役
- Azure portal：WebベースのGUI管理ツール
- Azure PowerShell：PowerShellコマンドレットでAzureを管理
- Azモジュール：Azure PowerShell用モジュール
- Azure CLI：`az` コマンドでAzureを管理
- Cloud Shell：ブラウザー上のコマンド実行環境
- Azure Mobile Apps：モバイル向けAzure管理アプリ
- ARMテンプレート：JSON形式でリソース構成を定義
- Azure Arc：ハイブリッド / マルチクラウドの一元管理
- SKU：リソースのラインナップや性能の種類
- 予約：長期利用向けの割引
- Azureハイブリッド特典：既存Microsoftライセンスを活用した割引
- Spot VM：余剰リソースを安く使うが停止・削除の可能性あり
- 料金計算ツール：Azure利用前の料金見積もり
- TCO計算ツール：オンプレミスからAzure移行時の総コスト比較
- コスト分析：現在発生しているAzure料金の確認
- 予算アラート：予算超過や予測超過を通知
- タグ：リソースやコストを分類する名前と値のセット
